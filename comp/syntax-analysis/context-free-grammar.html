Context-Free Grammars

The Formal Definition of a Context-Free Grammar​

一个 上下文无关文法 (CFG, Context-Free Grammar) GGG 是一个四元组 (T,NT,S,P)(T, NT, S, P)(T,NT,S,P)，其中：

  • TTT：终结符集 / 单词集 (Terminals / Terminal Symbols)，就是词法分析中的 token，是语言 L(G)L(G)L(G) 中的单词，终结符号是组成句子的基本单位。
  • NTNTNT：非终结符集 (Nonterminals / Variables)，是语法上的变量，表示了语言 L(G)L(G)L(G) 层次化的结构，出现在 GGG 的产生式中。
  • SSS：开始符 (Start Symbol)，单独从 NTNTNT 中指定一个非终结符作为开始符，SSS 所能生成的所有句子就是文法 GGG 代表的语言 L(G)L(G)L(G)。按照惯例，开始符的产生式总是写在第一个。
  • PPP：产生集 / 规则集 (Productions / Rules)，文法 GGG 中所有产生式的集合，CFG 的产生式具有 NT→(T∪NT)+NT \rightarrow (T \cup NT)^+NT→(T∪NT)+ 的形式，也就是说，产生式将一个非终结符替换为一个符号串。
巴科斯范式 (BNF)

上下文无关文法的传统表示是巴科斯范式 (BNF, Backus Normal Form)，又称为巴科斯-诺尔范式 (BNF, Backus-Naur Form)。 它是由美国著名计算机科学家 John Backus1 和丹麦著名计算机科学家 Peter Naur2 首先引入的用来描述计算机语言语法的符号集。

算术表达式文法 (Arithmetic Expression Grammar)
E→E+T ∣ E−T ∣ TT→T∗F ∣ T/F ∣ FF→( E ) ∣ id\begin{aligned} E &\rightarrow E + T \ | \ E - T \ | \ T \\ T &\rightarrow T * F \ | \ T / F \ | \ F \\ F &\rightarrow ( \ E \ ) \ | \ \textbf{id} \end{aligned}ETF​→E+T ∣ E−T ∣ T→T∗F ∣ T/F ∣ F→( E ) ∣ id​

其中，EEE 代表 expression，TTT 代表 term，FFF 代表 factor，id\textbf{id}id 代表 identifier。

文法的分类

美国著名语言学家、哲学家 Noam Chomsky 在 1956 年提出了 Chomosky Hierarchy，将形式语言分为四类：

GrammarLanguageAutomatonProduction Rules
Type 0Recursively Enumerable (Unrestricted)Turing Machineγ→α\gamma \rightarrow \alphaγ→α (no constraints)
Type 1Context-SensitiveLinear-Bounded Non-Deterministic Turing MachineαAβ→αγβ\alpha A \beta \rightarrow \alpha \gamma \betaαAβ→αγβ
Type 2Context-FreeNon-Deterministic Pushdown AutomatonA→αA \rightarrow \alphaA→α
Type 3RegularFinite State AutomatonA→aA \rightarrow aA→a
or A→aBA \rightarrow aBA→aB

符号的含义：

  • aaa = terminal
  • A,BA, BA,B = nonterminal
  • α,β,γ\alpha, \beta, \gammaα,β,γ = string of terminals and/or nonterminals
    • α,β\alpha, \betaα,β maybe empty
    • γ\gammaγ is never empty

从上下文有关文法的产生式规则中可以看出，只有当非终结符 AAA 在 α\alphaα 和 β\betaβ 的中间时 (或者说在 α\alphaα 和 β\betaβ 的上下文中)，才可以将 AAA 替换成非空的 γ\gammaγ。 而在上下文无关文法的产生式规则中，无论非终结符 AAA 的上下文是什么，我们总是可以将 AAA 替换成 α\alphaα。 这就是所谓的上下文有关与上下文无关。

对于正则文法的产生式规则，实际上表示的是，当有限自动机的输入为 α\alphaα 时，它将从状态 AAA 转移到状态 BBB。

对于无限制文法，超出了编译原理一般上研究的范畴，而且笔者对此了解不多，故不作解释，有兴趣的读者可以自行搜索或查阅相关资料。

在工程中，很多编程语言是上下文有关的，但为什么仍用 CFG 进行语法分析呢？

  • CFG 可以完美地描述表达式和语句的递归语法。
  • CSG 语法分析器效率低。
  • 编程语言大部分结构是 CF 的，剩下 CS 的部分可以在语义分析阶段进行分析。
    • 比如 if statement, declarations 等是 CF 的。
    • 比如 define before use, matching formal parameters 等问题是 CS 的。

Derivations​

将产生式看成是重写规则，产生式实际上在做的就是符号串的替换。

一个 句子 (Sentence) 指的是一个可以由文法 GGG 生成的符号串，其中所有的符号都是终结符。 一个文法 GGG 生成的 语言 (Language) L(G)L(G)L(G) 是它所有句子组成的集合。 如果两个文法生成的语言相同，这两个文法就被称为是等价的。

一个 推导 (Derivation) 指的是一个应用产生式规则的序列，以文法 GGG 的开始符作为开始，以 L(G)L(G)L(G) 中的一个句子作为结束。 我们用 α⇒β\alpha \Rightarrow \betaα⇒β 表示符号串 α\alphaα 推导出 符号串 β\betaβ (α\alphaα derives β\betaβ)。

  • α⇒β\alpha \xRightarrow{} \betaα​β: Derives in one step.
  • α⇒∗β\alpha \xRightarrow{*} \betaα∗​β: Derives in zero or more steps. Especially, α⇒∗α\alpha \xRightarrow{*} \alphaα∗​α (zero step).
  • α⇒+β\alpha \xRightarrow{+} \betaα+​β: Derives in one or more steps.

如果 S⇒∗αS \xRightarrow{*} \alphaS∗​α，其中 SSS 是文法 GGG 的开始符号，则称 α\alphaα 是 GGG 的一个 句型 (Sentential Form)。特别的，一个句子也是一个句型，句子是不包含非终结符的句型。

Note

一个句型可能既包含非终结符，又包含终结符，也可能是空串。

在推导中，我们可以在其中的每一步选择某个非终结符按照产生式进行替换，但这毕竟属于人工操作，而语法分析器需要自动化地进行解析，它会以固定的方式选择被替换的非终结符。

  • 最左推导 (Leftmost Derivation)：每步总是选择句型最左边的非终结符进行替换。用 α⇒lmβ\alpha \xRightarrow[lm]{} \betaαlm​β 表示。
  • 最右推导 (Rightmost Derivation)：每步总是选择句型最右边的非终结符进行替换。用 α⇒rmβ\alpha \xRightarrow[rm]{} \betaαrm​β 表示。
Example

考虑下面的文法：

E→E + E ∣ E ∗ E ∣ −E ∣ ( E ) ∣ idE \rightarrow E \ + \ E \ | \ E \ * \ E \ | \ -E \ | \ ( \ E \ ) \ | \ \textbf{id}E→E + E ∣ E ∗ E ∣ −E ∣ ( E ) ∣ id

和串：

−id∗(id+id)- \textbf{id} * ( \textbf{id} + \textbf{id})−id∗(id+id)

一个最左推导为：

E⇒E∗E⇒−E∗E⇒−id∗E⇒−id∗(E)⇒−id∗(E+E)⇒−id∗(id+E)⇒−id∗(id+id)\begin{aligned} E &\Rightarrow E * E \Rightarrow -E * E \Rightarrow -\textbf{id} * E \Rightarrow -\textbf{id} * (E) \\ &\Rightarrow -\textbf{id} * (E + E) \Rightarrow -\textbf{id} * (\textbf{id} + E) \Rightarrow -\textbf{id} * (\textbf{id} + \textbf{id}) \end{aligned}E​⇒E∗E⇒−E∗E⇒−id∗E⇒−id∗(E)⇒−id∗(E+E)⇒−id∗(id+E)⇒−id∗(id+id)​

一个最右推导为：

E⇒E∗E⇒E∗(E)⇒E∗(E+E)⇒E∗(E+id)⇒E∗(id+id)⇒E∗(id+id)⇒−id∗(id+id)\begin{aligned} E &\Rightarrow E * E \Rightarrow E * (E) \Rightarrow E * (E + E) \Rightarrow E * (E + \textbf{id}) \\ &\Rightarrow E * (\textbf{id} + \textbf{id}) \Rightarrow E * (\textbf{id} + \textbf{id}) \Rightarrow -\textbf{id} * (\textbf{id} + \textbf{id}) \end{aligned}E​⇒E∗E⇒E∗(E)⇒E∗(E+E)⇒E∗(E+id)⇒E∗(id+id)⇒E∗(id+id)⇒−id∗(id+id)​

我们在上面的例子中将 ⇒lm\xRightarrow[lm]{}lm​ 和 ⇒rm\xRightarrow[rm]{}rm​ 简写成 ⇒\Rightarrow⇒。 为了强调 α\alphaα 经过一个最左推导 (或最右推导) 得到 β\betaβ，我们写成 α⇒lm∗β\alpha \xRightarrow[lm]{*} \betaα∗lm​β (或 α⇒rm∗β\alpha \xRightarrow[rm]{*} \betaα∗rm​β)。

对于文法 GGG，它的开始符号是 SSS，如果有 S⇒lm∗αS \xRightarrow[lm]{*} \alphaS∗lm​α (或 S⇒rm∗αS \xRightarrow[rm]{*} \alphaS∗rm​α)， 我们称 α\alphaα 是文法 GGG 的一个 最左句型 (Left-Sentential Form) (或最右句型, Right-Sentential Form)。

最右推导有时也称为 规范推导 (Canonical Derivation)，在后面我们会看到，最右推导与自底向上语法分析相关。 最左推导则对应于自顶向下构造语法分析树的过程。由规范推导得到的句型称为 规范句型 (Canonical Sentential Form)。

Parse Trees​

一个 语法分析树 (Parse Tree) 是一个推导的图形表示，它过滤掉了推导的顺序信息。

  • 每个 内部节点 (interior node) 表示一个产生式的应用。
  • 叶子节点 (leaves) 既可以是非终结符，也可以是终结符。
    • 将语法分析树的叶子从左到右排列就可以得到一个句型，称为这棵树的 产出 (yield) 或 边缘 (frontier)。

Ambiguity and Its Elimination​

对于一个上下文无关文法 GGG，如果 GGG 存在一个句子有两种或两种以上的语法分析树 (或最左推导)， 那么称文法 GGG 是 二义性文法 (Ambiguous Grammar)。

与之相对的，如果文法 GGG 的每个句子都只有唯一的语法分析树 (或最左推导)， 那么称文法 GGG 是 非二义性文法 (Ambiguous Grammar)。

空悬 else (Dangling Else)

Dangling Else Grammar:

stmt→ if expr then stmt  ∣if expr then stmt else stmt  ∣other\begin{aligned} \textit{stmt} &\rightarrow \ \textbf{if} \ \textit{expr} \ \textbf{then} \ stmt \\ &\ \ | \quad \textbf{if} \ \textit{expr} \ \textbf{then} \ \textit{stmt} \ \textbf{else} \ \text{stmt} \\ &\ \ | \quad \textbf{other} \end{aligned}stmt​→ if expr then stmt  ∣if expr then stmt else stmt  ∣other​

对于下面这个复合的条件语句：

if E1 then S1 else if E2 then S2 else S3\textbf{if} \ E_1 \ \textbf{then} \ S_1 \ \textbf{else} \ \textbf{if} \ E_2 \ \textbf{then} \ S_2 \ \textbf{else} \ S_3if E1​ then S1​ else if E2​ then S2​ else S3​

消除二义性的方法：

  • 指定优先级 (Specify precedence)
    • The higher level of the production, the lower priority of operator.
    • The lower level of the production, the higher priority of operator.
  • 指定结合性 (Specify associativity)
    • If the operator is left associative, induce left recursion in its production.
    • If the operator is right associative, induce right recursion in its production.

Is there any algorithm or method that can check whether a grammar is ambiguous?3
